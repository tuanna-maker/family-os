STOS Family OS — Giới thiệu & Kiến trúc

1. Giới thiệu & Phạm vi tài liệu

Tài liệu này là Đặc tả Yêu cầu Phần mềm (SRS) đầy đủ của hệ sinh thái STOS Family OS — bao gồm ứng dụng STOS Family (cư dân) và STOS Guard (bảo vệ) chạy trên iOS và Android, xây dựng trên Capacitor 6 + React 19 + Supabase.

SRS này bao gồm: Yêu cầu chức năng (FR), Chi tiết Use Case (mục 4A) theo chuẩn TASMOS SRS v3.0, yêu cầu phi chức năng (NFR), database schema, API reference, navigation map và chiến lược kiểm thử. Tài liệu nguồn luồng nghiệp vụ: UNICOM/BRD-FAMILYOS-001 v1.2. · UC v1.2.

4. Yêu cầu chức năng (FR)

FR-01: Authentication & Authorization

Mã Yêu cầu Độ ưu tiên
FR-01-001 Đăng nhập bằng email/password qua Supabase Auth. Session JWT lưu trong @capacitor/preferences (không phải localStorage trên native). P0 — Bắt buộc
FR-01-002 Hỗ trợ đăng nhập bằng username: tìm email từ username → gọi signInWithPassword (username.functions.ts). P0 — Bắt buộc
FR-01-003 Sau đăng nhập, resolve-destination.ts định tuyến: role family_* → /home (Family App); role security_* → /guard (Guard App). P0 — Bắt buộc
FR-01-004 Forgot Password và Reset Password qua email magic link. Deep link callback: vn.unicom.stos.family://auth và vn.unicom.stos.guard://auth. P0 — Bắt buộc
FR-01-005 Mỗi route có beforeLoad guard kiểm tra session. Session hết hạn → redirect /login?redirect=current_path. P0 — Bắt buộc
FR-01-006 RBAC 4 roles: family_owner, family_member, security_guard, security_supervisor. Mỗi bảng DB có RLS policy theo role. P0 — Bắt buộc

FR-02: Dashboard & Gia đình

Mã Yêu cầu Độ ưu tiên
FR-02-001 Màn hình Home (/home): trạng thái an ninh (chip màu xanh/vàng/đỏ), thành viên gia đình online, shortcut 8 tính năng chính, lịch hôm nay. P0
FR-02-002 BottomNav cố định 5 tab: Home · Gia đình · Bảo an · Cộng đồng · Tài khoản. SideNav (swipe left) liệt kê tất cả modules. P0
FR-02-003 Màn Gia đình (/gia-dinh): danh sách thành viên, avatar, vai trò, quick-action đến module liên quan từng thành viên. P0

STOS Family OS — API Reference

6. API Reference

6.1. Supabase Client Functions

Function Operation Bảng Mô tả
listExpenses({family_id, from, to}) SELECT expenses Chi tiêu gia đình theo khoảng thời gian. RLS tự lọc family_id.
createExpense({family_id, title, amount, category, spent_on}) INSERT expenses Tạo khoản chi. Validation: amount > 0, title không rỗng.
createSecurityRequest({type, family_id, description}) INSERT security_requests Gửi SOS/yêu cầu an ninh. Trigger push notification Guard.
updateSecurityRequest({id, status}) PATCH security_requests Bảo vệ cập nhật: open → in_progress → resolved.
listFamilyEvents({family_id}) SELECT family_events Tất cả sự kiện gia đình. Filter ngày/loại ở client.
upsertFamilyEvent({id?, family_id, title, category, starts_at, ...}) UPSERT family_events Tạo mới (không id) hoặc cập nhật (có id) sự kiện.
listMedicineReminders({elderly_profile_id}) SELECT medicine_reminders Lịch thuốc ông bà theo ngày.
logMedicineTaken({reminder_id, taken_at}) INSERT medicine_logs Ghi nhận đã uống thuốc.
getSecurityStatus({family_id}) SELECT security_requests Tổng hợp chip an ninh tòa nhà cho màn Home.
getFamilyToday({family_id}) SELECT families, children Dữ liệu tổng hợp hôm nay cho Dashboard.
listNotifications({user_id}) SELECT notifications Thông báo in-app của user. Mark as read.

6.2. Supabase Edge Functions

Function Method Auth Mô tả chi tiết
scan-receipt POST JWT User Nhận {image_base64} → gọi OpenAI GPT-4 Vision → trả {title, amount, category, date}. Zod validation input/output.
log-ingest POST Anon/JWT Batch nhận [{level, message, app, session_id, ts}] từ mobile → INSERT app_logs. Strip PII (email/phone) trước khi lưu.
health-check GET Service role Cron mỗi 1 phút: probe DB + auth → INSERT health_checks. Alert nếu latency > ngưỡng.
metrics-aggregate POST Service role Cron mỗi 5 phút: gọi RPC refresh_metrics_views() → cập nhật metrics_hourly.
